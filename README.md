## 🎯 Резюме и Цели

### **Предназначение**
Основната цел на **ActPro** е да дигитализира процеса на свързване между собствениците на спортни съоръжения и активните хора. Системата решава проблема с липсата на централизирана платформа, където потребителите могат да откриват, сравняват и резервират спортни активности в реално време.

### **Анализ на потребностите**
В момента много спортни комплекси разчитат на остарели методи (тефтери, Excel), което води до:
* **Чести грешки** и дублиране на резервации.
* **Липса на прозрачност** за крайния клиент относно свободните часове.
* **Невъзможност за обратна връзка** и структурирани ревюта.

## 🛠️ Техническа Реализация

### **Технологичен стек**
* **Език за програмиране:** C# (.NET 9.0)
* **Среда за разработка:** Microsoft Visual Studio 2022
* **База данни:** Microsoft SQL Server
* **ORM:** Entity Framework Core (Code-First approach)
* **Frontend:** Bootstrap 5, Razor Views, JavaScript/jQuery
* **Библиотеки:** MailKit (за имейл комуникация), Leaflet.js (за интерактивна карта)

### **Архитектура**
Приложението е изградено по модела **MVC (Model-View-Controller)**, разделяйки логиката на данни, интерфейс и управление. Използван е **Repository Pattern** заедно със **Service Layer** за по-добра абстракция.

---

## 🚀 Ниво на сложност и Решени проблеми

По време на разработката бяха имплементирани сложни алгоритми и защити:
1.  **Concurrency Control:** Алгоритъм за предотвратяване на дублирани резервации в един и същ часови слот.
2.  **Мрежова устойчивост:** Асинхронни процеси (`async/await`) при работа с външни SMTP сървъри.
3.  **Цялост на данните:** Управление на сложни връзки (**Foreign Keys**) и каскадно изтриване при премахване на обекти с активна история.
4.  **Одит и сигурност:** Система за **Audit Logs**, записваща всяко критично действие, и защита срещу SQL Injection и XSS атаки.

## 👤 Потребителски Роли

* **Гост:** Разглеждане на обекти и начална страница.
* **Регистриран потребител:** Резервации в реално време, история на посещенията и оставяне на ревюта.
* **Собственик (Business Dashboard):** Управление на обекти, снимки, графици и статистика на приходите.
* **Администратор:** Пълен контрол над съдържанието, одобрение на обекти, управление на роли и преглед на системни логове.

## 📝 Инсталиране и стартиране

1.  Уверете се, че имате инсталиран **.NET Runtime** и **SQL Server**.
2.  Клонирайте репозиторито.
3.  Конфигурирайте настройките за връзка в `appsettings.json`.
4.  Изпълнете автоматичните миграции:
    ```bash
    dotnet ef database update
    ```
5.  Стартирайте проекта чрез Visual Studio или `dotnet run`.
